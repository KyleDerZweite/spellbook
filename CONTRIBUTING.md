# Contributing

Thanks for your interest in improving Spellbook.

## Before You Start

1. Open an issue for bugs, feature proposals, or significant changes.
2. Check existing issues and PRs to avoid duplicate work.
3. Keep changes focused and small.

## Development Workflow

1. Fork the repository and create a branch from `v1`.
2. Make your changes with clear commit messages.
3. Run relevant tests and linters locally before opening a PR.
4. Open a pull request against `v1` with a concise description.

## Pull Request Expectations

1. Explain what changed and why.
2. Reference related issues.
3. Include screenshots for UI changes when helpful.
4. Keep PR scope limited to one concern whenever possible.

## Security

Do not open public issues for vulnerabilities. See `.github/SECURITY.md` for private reporting instructions.
